# 📊 Lab 5 – Feature Engineering (Classification)

This lab focuses on **feature engineering for a classification problem** using a Talabat-style food delivery dataset.  
The objective is to predict the **Order_Status** of each order (*Delivered, Cancelled, In Transit*) by engineering meaningful features and training a baseline machine learning model.

The dataset used in this lab is already **clean**, meaning there are **no missing values or duplicate rows**, allowing us to focus entirely on **feature engineering and model interpretation**.

## 🎯 Lab Objective

The goal of this lab is to understand how **feature engineering affects machine learning performance** in a classification task.

The model predicts the **order status** based on information available early in the order lifecycle.

Possible order statuses:

- **Delivered**
- **Cancelled**
- **In Transit**

---

## ⚙️ What Was Done in This Lab

The workflow of this lab follows a typical **machine learning pipeline**:

1. Dataset loading and inspection  
2. Target variable analysis and class distribution visualization  
3. Feature engineering  
4. Feature preprocessing and encoding  
5. Training a baseline **Random Forest classifier**  
6. Evaluating model performance  
7. Interpreting **feature importance**
